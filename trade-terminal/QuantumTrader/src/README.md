QuantumTrader: Структура проекта и Дорожная карта (Roadmap)
📁 Архитектура проекта (Directory Structure)
Проект строго разделен на бизнес-логику (Core) и пользовательский интерфейс (UI) для соблюдения принципов чистого кода и SOLID.

🗺️ Дорожная карта разработки (Roadmap)
Этап 1: Архитектура оконного менеджера и UI
Умная сетка графиков (Grid Layout & Window Management)

Алгоритм автоматического распределения новых док-окон (QDockWidget).

Графики открываются строго слева направо. При достижении лимита ширины (например, < 600px) окно переносится на новую строку вниз.

Защита от переполнения: Если экран полностью заполнен сеткой, новые графики автоматически накладываются поверх старых в виде вкладок (Tabbed View), начиная с левого верхнего угла.

Контекстное меню графиков (UI Skeleton)

Переопределение события contextMenuEvent в FastChart.

Создание стильного выпадающего меню (QMenu) по правому клику мыши.

Базовые пункты: "Таймфреймы" (M1, M5, H1 и т.д.), "Индикаторы", "Свойства графика" (пока без глубокой логики, только UI и qDebug для проверки кликов).

Этап 2: Торговое ядро и Аналитика
Движок Таймфреймов (Timeframes Core)

Математический модуль для перестроения свечей "на лету".

Алгоритм агрегации: программная склейка базового массива данных (тиков или минуток) в свечи старших таймфреймов (Open, High, Low, Close) с последующей отрисовкой.

Базовые индикаторы теханализа (C/C++ Интеграция)

Внедрение классических инструментов (RSI, MACD, SMA/EMA).

Подключение сверхбыстрой индустриальной C-библиотеки (например, TA-Lib).

Написание C++ оберток для бесшовной передачи массивов свечей в библиотеку и возврата точек для рендеринга.

Пользовательские скрипты (Lua Engine)

Интеграция интерпретатора Lua в C++ движок терминала (через sol2).

Проброс C++ классов (свечи, функции рисования OpenGL) в среду Lua.

Выполнение пользовательских .lua скриптов из папки программы "на лету" без перекомпиляции ядра терминала.

Этап 3: Система динамической кастомизации (QSS Editor)
Интеграция редактора в экосистему ADS

Перенос собственного редактора стилей в полноценную док-панель.

Обеспечение персистентности: сохранение написанного кода при скрытии панели.

"Живое" применение глобальных стилей (Global Scope) ко всему интерфейсу без перезагрузки.

Контекстное управление стилями (Local Styling)

Внедрение системы селекторов по уникальным ID окон (например, #Chart_EURUSD_5M).

Переключатель области видимости (Scope Selector): выбор между всем приложением и конкретным рабочим пространством.

Изоляция стилей: возможность изменить фон или сетку одного графика, не затрагивая остальные.

Интеллектуальная линковка (Link Groups)

Создание системы цветовых групп (Red, Blue, Green) для синхронизации инструментов.

Применение уникальной визуальной CSS-схемы сразу ко всей выбранной группе графиков.

Пространственная привязка (DockArea Binding)

Механизм "соседства": стилизация всех вкладок, находящихся в одном физическом контейнере с редактором.

Оптимизация UX для мульти-мониторных конфигураций (разные визуальные пресеты для разных экранов для снижения нагрузки на зрение).

Твое замечание про индикаторы — это фундаментальная истина. Если мы без спроса поменяем человеку 1-минутный график на 1-дневный, его скользящая средняя (SMA 200) просто сломается, а торговый робот выдаст ложные сигналы. Так делать категорически нельзя.

Давай я объясню эти два понятия еще раз, но уже в контексте суровой реальности торговых терминалов.

1. Как на самом деле работает LOD (Level of Detail) в трейдинге
Ты абсолютно прав: менять таймфрейм (склеивать свечи) можно только по явному запросу пользователя (когда он сам выбирает "5М" или "1H" в меню).

Но что делать, если он остался на 1-минутном таймфрейме, но отзумил график так сильно, чтобы увидеть весь год? Монитор имеет ширину, скажем, 1920 пикселей. А свечей за год — 500 000. На один пиксель экрана попадает по 260 свечей.

Если заставить видеокарту рисовать 260 свечей в одной и той же точке экрана, она будет просто закрашивать один и тот же пиксель 260 раз подряд. Это убьет производительность (Overdraw).

Как это решается визуально (без изменения таймфрейма):
Терминал берет эти 260 минутных свечей, которые попали в один пиксель по ширине, математически находит среди них самую высокую цену (High) и самую низкую (Low) и рисует в этом столбце экрана всего одну вертикальную линию.
Всё! Визуально форма графика идеальная, индикаторы под капотом всё еще считаются по минуткам, а видеокарта рисует всего 1920 линий вместо 500 000 полигонов. Это и есть правильный LOD для трейдинга.
