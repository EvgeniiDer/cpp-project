🗺️ Дорожная карта (Roadmap) разработки QuantumTrader
1. Умная сетка графиков (Grid Layout & Window Management)

Описание: Настроить алгоритм автоматического распределения новых окон (QDockWidget) в главном окне терминала.

Логика работы (Алгоритм):

Новые графики открываются строго слева направо.

Если по ширине экрана достигнут лимит (например, окно меньше 600px) — следующий график переносится на "новую строку" вниз.

Если экран полностью заполнен плиткой и по ширине, и по высоте — включается "защита от тестировщика": новые графики начинают накладываться поверх старых в виде вкладок (Tabbed View), начиная с верхнего левого угла по кругу.

2. Визуальный шаблон контекстного меню (UI Заглушка)

Описание: Создать интерфейс меню по клику правой кнопкой мыши внутри окна графика (FastChart).

Логика работы:

Переопределить событие contextMenuEvent.

Сверстать стильное выпадающее меню (через QMenu).

Добавить пункты: "Таймфреймы" (с подменю M1, M5, H1 и т.д.), "Индикаторы" и "Свойства графика".

Важно: На этом этапе реализация логики не требуется, только визуал и вывод сообщений в консоль (qDebug), чтобы убедиться, что клики фиксируются.

3. Ядро Таймфреймов (Timeframes Core)

Описание: Написать математический движок для перестроения свечей на графике.

Логика работы: * Привязка кнопок из контекстного меню к функции обновления данных.

Реализация алгоритма агрегации: брать базовый массив данных (например, тики или минутные свечи) и программно "склеивать" их в свечи нужного таймфрейма (Open, High, Low, Close) с последующей перерисовкой графика.

4. Базовые встроенные индикаторы (C-библиотека)

Описание: Внедрить классические инструменты технического анализа (RSI, MACD, SMA/EMA и т.д.).

Логика работы:

Не изобретать велосипед, а подключить сверхбыструю индустриальную библиотеку на языке C (например, TA-Lib).

Написать C++ обертку, которая будет передавать массив наших свечей в функции библиотеки и получать готовые массивы точек для отрисовки линий поверх графика.

5. Пользовательские скрипты / Индикаторы (Lua Engine)

Описание: Дать пользователям возможность писать собственные торговые алгоритмы и индикаторы без перекомпиляции самого терминала.

Логика работы:

Встроить интерпретатор языка Lua в C++ движок (используя библиотеку вроде sol2).

Пробросить C++ классы (свечи, функции рисования) в среду Lua.

Сделать так, чтобы терминал умел подхватывать текстовые файлы .lua из папки программы, исполнять их на лету и выводить результат на график.
